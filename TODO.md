# TODO

TODO 是 "to do" 的缩写，意思是待办事项清单。它用来记录接下来要做什么、哪些已经完成、哪些被阻塞，避免项目推进时任务散落在聊天记录或脑子里。

## 使用方式

- `[ ]` 表示尚未完成。
- `[x]` 表示已经完成。
- 每次论文写作、代码实验或资料整理后，都应更新相关任务状态。
- 论文写作细节任务放在 `docs/MPD/TODO.md`，项目级任务放在本文件。

## 项目级任务

- [x] 初始化本地 Git 仓库。
- [x] 建立项目目录结构。
- [x] 建立 README。
- [x] 建立 TODO 清单。
- [x] 放入 MPD Word 主文档。
- [ ] 成功推送到 GitHub 远端仓库。
- [ ] 补充论文研究问题和核心贡献。
- [ ] 整理第一批相关论文到 `docs/Papers/`。
- [ ] 明确数据集来源、权限和预处理方案。
- [ ] 设计第一版检索模块实验。
- [ ] 设计第一版扩散模型实验。
- [ ] 确定评价指标和对照实验。
